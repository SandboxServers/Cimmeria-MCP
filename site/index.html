🔍 RAG Search Tools

search_cimmeria
Semantic search across the Cimmeria server, SGW client, and BigWorld engine codebases. Returns code, docs, configs, and UI scripts ranked by relevance. Routes cimmeria-server queries through Azure AI Search (hybrid HNSW + BM25); falls back to Cosmos DB vector search for sgw-client and bigworld-engine.
"Search the cimmeria-server source for how entity replication works"
list_cimmeria_files
List all indexed file paths, optionally filtered by extension or source project. Useful for discovering what's in the index before searching.
"List all the Rust files indexed from the cimmeria-server"
get_file_content
Retrieve the full content of a specific file by path. Reassembles all chunks in order for multi-chunk files.
"Show me the full content of scripts/cell/SGWPlayer.py"
find_similar_code
Find code patterns similar to a given snippet using pure vector search. Good for finding parallel implementations across Python, C++, and Rust.
"Find code similar to this damage calculation: fn handle_damage(&mut self, amount: f32)"
get_project_overview
Get file counts by type, directory tree, total indexed chunks, and available source projects. The bird's-eye view of the index.
"Give me an overview of the BigWorld engine codebase"
search_by_directory
Semantic search scoped to a specific directory path prefix. Combines text + vector relevance within a directory subtree.
"Search for combat-related code inside the scripts/cell/ directory"

🕸 Knowledge Graph Tools

get_entity_details
Full details about an SGW entity or interface — all properties with types and flags, all methods by type (Client/Cell/Base), parent entity, implemented interfaces, and game system links.
"What are all the properties and methods on the SGWPlayer entity?"
get_inheritance_tree
Full inheritance hierarchy — ancestors (who this entity extends), descendants (who extends it), and all implemented interfaces up the chain.
"Show me the full inheritance hierarchy for SGWMob"
get_graph_overview
Overview of the entire knowledge graph — vertex/edge counts by type, list of all entities, interfaces, and game systems.
"Give me an overview of the Cimmeria knowledge graph"
get_game_system_details
Details about a game system and its associated entities. Systems: combat, inventory, missions, chat, spawning, gate_travel, organizations, trading, groups, account, minigame, cover.
"What entities are involved in the combat system?"
get_replicated_properties
All client-visible (CELL_PUBLIC) properties for an entity, including inherited properties from the full inheritance chain and implemented interfaces. Essential for client compatibility.
"What properties does the client see for SGWPlayer?"
get_method_call_chain
Trace what a method calls outward (client RPCs, base/cell messages, BigWorld APIs, other methods) and what calls it inward. Shows full data flow.
"What does SGWMob.threatGenerated call, and what calls it?"
traverse_graph
Walk the graph from a starting entity following a specific edge type to configurable depth. Edge types: inherits, implements, has_property, has_client_method, has_cell_method, has_base_method, part_of_system, sends_to_client, calls_self, and more.
"What interfaces does SGWPlayer implement? Traverse 2 levels deep."
lookup_enum
Look up SGW enumerations by name (returns all tokens/values) or search by token name (returns which enum it belongs to). 124 enumerations + 1,276 constants.
"What are all the values in the EDamageType enumeration?"
resolve_type
Resolve a BigWorld type alias to its full structure. Handles simple aliases (CONTROLLER_ID → INT32), arrays, and FIXED_DICT compound types with all fields. 63 type aliases available.
"What fields are in the CharacterInfo FIXED_DICT type?"
lookup_game_def
Game data definition schemas — fields, cross-references to other defs, and enum references. Covers Ability, Effect, Item, Mission, LootTable, Dialog, Stargate, EntityTemplate, Container, Discipline, and more.
"What's the schema for the Ability game definition?"
get_implementation_status
Compare .def method definitions against Python and C++ implementations. Without entity_name: overview of all entities. With entity_name: per-method breakdown showing which are defined, Python-implemented, and C++-implemented.
"Which methods on SGWPlayer have Python implementations vs just .def definitions?"
cross_reference
Search across the entire knowledge graph for a term. Finds matching methods (def/Python/C++), properties, enumerations, constants, game definitions, and call chains. One query, full picture.
"Cross-reference everything in the graph related to 'Alignment'"
get_entity_protocol
Complete client-server protocol contract — all ClientMethods (server→client), exposed BaseMethods (client→server), CellMethods (gameplay), and auto-replicated CELL_PUBLIC properties. Includes inherited methods from the full chain.
"What's the full client-server protocol for the Account entity?"
lookup_bigworld_api
BigWorld engine API usage across the Python codebase. Without api_name: lists all APIs with usage counts. With api_name: shows which methods call it and any C++ reimplementations.
"Which entities use BigWorld.addTimer and how?"

✨ AI Skill Tools

explain_cimmeria
Ask a question about the codebase and get an AI-synthesized answer. Gathers RAG search results + knowledge graph context, then uses GPT-5.1 Chat to produce a coherent explanation with sources and confidence rating.
"How does the combat system handle threat generation between mobs and players?"
generate_entity_stub
Generate a Rust struct/impl stub from an entity's .def definition. Maps BigWorld types to Rust, groups methods into impl blocks by type, derives traits from interfaces, and uses composition for parent entities.
"Generate the Rust struct and impl stubs for SGWPlayer"
translate_python_to_rust
Convert Python BigWorld entity scripts to idiomatic Rust. Translates BigWorld API calls, entity communication patterns, and Python idioms while preserving behavioral logic exactly. Can target a full entity or a single method.
"Translate the SGWMob.doAiAction method from Python to Rust"
generate_tests
Generate Rust #[test] functions derived from Python behavior, .def contracts, and protocol requirements. Covers unit, property, protocol, and integration tests.
"Generate Rust tests for SGWPlayer.takeDamage"
troubleshoot
Diagnose issues with Cimmeria server code. Checks protocol mismatches, implementation gaps, type mismatches, missing replicated properties, incorrect method signatures, and BigWorld API behavioral differences.
"The client disconnects whenever a player tries to trade with another player"
review_code
Review Rust code against original SGW .def specs and Python behavior. Checks protocol correctness, property completeness, behavioral fidelity, type safety, and Rust idioms. Issues rated CRITICAL / WARNING / INFO.
"Review this Rust implementation of SGWPlayer against the original spec"
check_compatibility
Verify Rust code against the fixed SGW client binary. The client CANNOT be modified — checks every ClientMethod, exposed BaseMethod, CELL_PUBLIC property, and argument type. Returns a compatibility score.
"Will this SGWMob Rust implementation work with the original game client?"
analyze_impact
Analyze the impact of changing a method or property. Traces all dependents through call chains, cross-entity references, client visibility, and inheritance. Rates cascade risk LOW / MEDIUM / HIGH / CRITICAL.
"What would break if I changed the hitPoints property on SGWBeing?"
plan_implementation
Create a detailed Rust implementation plan: struct definition, traits, method priority (client-critical first), dependencies, data flow, Python behavioral notes, test strategy, and suggested file structure.
"Create an implementation plan for SGWMob in Rust"
whats_next
Recommend what to implement next by analyzing all entities' implementation status. Ranks by client-criticality, dependency count, Python coverage, and game system impact.
"What entity should I implement next for the Cimmeria server?"
analyze_protocol
Analyze the client-server protocol for an entity or game system. Produces a message catalog, data contracts, sequence diagrams, critical compatibility notes, and serialization requirements.
"Analyze the full client-server protocol for the combat system"
trace_sequence
Trace the complete message sequence for a game scenario with step-by-step method calls, direction and data, branching paths, client updates, and timing.
"Trace what happens step by step when a player loots a mob"
generate_diagram
Generate Mermaid diagrams — class, sequence, flowchart, state, or dependency graphs. Output is renderable in any Mermaid-compatible tool.
"Generate a class diagram showing SGWPlayer's inheritance tree"
decode_game_design
Reverse-engineer the game design of an SGW feature from its code. Reconstructs gameplay loops, rules and mechanics, progression, system interactions, and data models. Written for game designers, not programmers.
"What's the game design behind the gate travel system?"
